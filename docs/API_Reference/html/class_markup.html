◆ addRichText()

Markup::addRichText (   datas)
inline

Add rich text string to the end, including rich text strings and styles.

Note: Now, only support Freetext annotation. if current annotation is a freetext and its intent is "FreeTextTypewriter", text aligment of input style will be set to all rich text strings of current freetext annotation.

Parameters
datas{Array[{data}]} - An array of rich text data.
data{Object} - rich text data object.
data.content{string} - Text string to be set as content for specified rich text. This should not be an empty string.
data.richTextStyle{object} - The style data of a rich text string
data.richTextStyle.font{object} - A valid font object.
data.richTextStyle.font.name{string} - the face name of the font.
data.richTextStyle.font.styles{number} - The font styles. Please refer to Font_Styles .
data.richTextStyle.font.charset{number} - The charset of the font. Please refer to Font_Charset or Font_CIDCharset .
data.richTextStyle.textSize{number} - Text size. It should not be negative value. 0 means text will not be shown.
data.richTextStyle.textAlignment{number} - Alignment value. Please Refer to Alignment .
data.richTextStyle.textColor{number} - Text color. Format: 0xRRGGBB.
data.richTextStyle.isBold{boolean} - A boolean value which indicates whether to make text bold or not.
data.richTextStyle.isItalic{boolean} - A boolean value which indicates whether to italicize text or not.
data.richTextStyle.isUnderline{boolean} - A boolean value which indicates whether to underline text or not.
data.richTextStyle.isStrikethrough{boolean} - A boolean value which indicates whether to cross text out with strike through or not.
data.richTextStyle.cornerMarkStyle{number} - Corner mark style which can be used to make text as superscript or subscript or not as any kind of corner mark. 1 - Corner mark style: none. 2 - Corner mark style: superscript. 3 - Corner mark style: subscript.
Since
8.4.0

◆ getIntent()

Markup::getIntent ( )
inline

Get intent name.
Following annotation types have predefined intent name:

  • FreeText annotation's predefined intent name: "FreeTextCallout", "FreeTextTypewriter". If no intent name is used, free text annotation would be shown as a text box.
  • Line annotation's predefined intent name: "LineArrow", "LineDimension". If no intent name is used, line annotation would be shown as a common line.
  • Polygon's predefined intent name: "PolygonCloud", "PolygonDimension". If no intent name is used, polygon annotation would be shown as a common polygon.
  • PolyLine annotation's predefined intent name: "PolyLineDimension". If no intent name is used, polyline annotation would be shown as a common polyline.
    Returns
    {string} - Annotaions's intent.

Markup::insertRichText (   datas)
inline

Insert rich text string to the place which is specified by indexs.

Note: Now, only support Freetext annotation. if current annotation is a freetext and its intent is "FreeTextTypewriter", text aligment of input style will be set to all rich text strings of current freetext annotation.

Parameters
datas{Array[{data}]} - An array of rich text data.
data{Object} - rich text data object.
data.index{number} - Index for a rich text string whose style data is to be inserted at. Valid range: from 0 to (count-1). count is returned by function getRichText. If input value is equal to rich text string count, that means to add the new rich text string to the end. In this case, this funcion equals to function AddRichText.
data.content{string} - Text string to be set as content for specified rich text. This should not be an empty string.
data.richTextStyle{object} - The style data of a rich text string
data.richTextStyle.font{object} - A valid font object.
data.richTextStyle.font.name{string} - the face name of the font.
data.richTextStyle.font.styles{number} - The font styles. Please refer to Font_Styles .
data.richTextStyle.font.charset{number} - The charset of the font. Please refer to Font_Charset or Font_CIDCharset .
data.richTextStyle.textSize{number} - Text size. It should not be negative value. 0 means text will not be shown.
data.richTextStyle.textAlignment{number} - Alignment value. Please Refer to Alignment .
data.richTextStyle.textColor{number} - Text color. Format: 0xRRGGBB.
data.richTextStyle.isBold{boolean} - A boolean value which indicates whether to make text bold or not.
data.richTextStyle.isItalic{boolean} - A boolean value which indicates whether to italicize text or not.
data.richTextStyle.isUnderline{boolean} - A boolean value which indicates whether to underline text or not.
data.richTextStyle.isStrikethrough{boolean} - A boolean value which indicates whether to cross text out with strike through or not.
data.richTextStyle.cornerMarkStyle{number} - Corner mark style which can be used to make text as superscript or subscript or not as any kind of corner mark. 1 - Corner mark style: none. 2 - Corner mark style: superscript. 3 - Corner mark style: subscript.
